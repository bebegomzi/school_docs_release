# 곰지의 문서 정리기

[![Latest release](https://img.shields.io/github/v/release/conoms/school_docs_release?label=latest&color=3651d4)](https://github.com/conoms/school_docs_release/releases/latest)
![Windows](https://img.shields.io/badge/Windows-10%20%7C%2011-0078D4)
![Local processing](https://img.shields.io/badge/document%20processing-local-1D9E75)

**교사가 학교 현장에서 자주 하는 PDF·문서 작업을 한 프로그램에서 처리하기 위한 Windows 데스크톱 도구입니다.**

PDF 편집과 서명, 개인정보 가리기, 분할·생성, 암호 관리, 일괄 인쇄, 파일명 변경, 얼굴 모자이크를 지원합니다. 문서 분석과 OCR은 사용자의 PC에서 처리하며, 불러온 PDF와 명단을 외부 서버로 전송하지 않습니다.

> 이 저장소는 일반 사용자를 위한 **공개 배포 전용 저장소**입니다.
>
> 설치 파일, 체크섬, 버전별 변경 사항은 [Releases](https://github.com/conoms/school_docs_release/releases)에서 확인할 수 있습니다.
>
> **v4.3.0부터 Conoms 공동 저장소에서 개발·배포합니다.** 기존 `bebegomzi` 배포 저장소의 v4.2.8은 이전 설치본을 이 저장소로 연결하는 마지막 이주용 버전입니다.

## 현재 개발 상태

| 항목 | 상태 |
| --- | --- |
| 최신 정식 버전 | **v4.3.0** |
| 지원 환경 | Windows 10·11 x64 |
| 배포 형태 | 설치 프로그램 없이 압축을 풀어 실행하는 portable 앱 |

공개 배포 저장소는 **v4.2.3부터** 운영합니다. 이전 버전에서 v4.2.3으로 이동할 때는 새 압축 파일을 한 번 직접 받아야 하며, 이후 버전부터는 프로그램 안에서 업데이트할 수 있습니다.

- ✅ PDF 편집·서명·페이지 편집 기본 기능 안정화

- ✅ 복합 양식 대응을 위해 Tesseract에서 PaddleOCR PP-OCRv5로 전환

- ✅ v4.2.3부터 버전 표시, 수동 확인, 제자리 업데이트와 실패 시 자동 복구 지원
- ✅ 학교생활기록부·재직증명서·주민등록표 등본·초본 양식 분석, 학생별 분리 저장과 파일명 복원 지원

## v4.3.0 핵심 개선

- 공식 소스와 공개 배포 저장소를 Conoms 공동 개발 조직으로 이전했습니다.
- 학교생활기록부의 일반·정부24 양식을 함께 감지하고, 신청인·학생 구간·반복 머리글을 양식에 맞게 분석합니다.
- 주민등록표 등본·초본을 자동 감지하고, 표의 항목별 개인정보 후보를 검토할 수 있습니다.
- 순수 스캔 PDF의 OCR 재분석 안내, 분석 없이 수동 편집, 일정 쪽수 간격 반복 마스킹을 지원합니다.
- 개인정보 편집창에서 진단 정보를 저장할 수 있으며 분석·저장 진행 상황을 단계와 백분율로 표시합니다.

### v4.2.8 이주 브리지

- 기존 `bebegomzi` 배포 저장소가 Conoms의 새 배포 저장소를 확인하도록 업데이트 경로를 전환했습니다.
- v4.2.7 이하 사용자는 먼저 v4.2.8로 업데이트한 뒤 v4.3.0으로 한 번 더 업데이트하면 이후 버전을 계속 받을 수 있습니다.

### v4.2.7 핵심 개선

- Windows 다운로드 차단 자동 해제 범위를 `_internal`과 검증된 앱 관리 파일로 제한해, 설치 폴더에 사용자가 따로 둔 문서의 보안 표시는 건드리지 않습니다.
- 수동 복구용 `school_docs 실행 오류 해결.cmd`도 같은 범위만 처리하도록 수정했습니다.

### v4.2.6 핵심 개선

- 개인정보 가리기에 **즐겨찾기 양식**을 추가했습니다. 학교생활기록부 I·II를 하나의 학교생활기록부 양식으로 분석하며 재직증명서도 지원합니다.
- 학교생활기록부 묶음을 학생별로 구분해 검토하고, 새 파일로 분리 저장할 수 있습니다. 파일명은 무작위로 섞거나 `5자리 학번_학생명`으로 유지할 수 있습니다.
- 무작위 파일명 저장 시 대응표를 만들며, 이름 변경 탭에서 파일 또는 폴더와 대응표를 불러 원래 학번·이름으로 복원할 수 있습니다.
- 분석 사전 점검과 저장 단계의 현재 쪽수·백분율·작업 단계를 표시합니다.
- Windows 다운로드 차단 때문에 실행이 실패하는 환경을 위해 시작 시 설치 폴더 차단 해제를 시도하고, 수동 복구용 `school_docs 실행 오류 해결.cmd`도 함께 제공합니다.
- 개인정보 가리기 초기화 오류로 `분석` 버튼이 반응하지 않던 문제를 수정하고, 한 화면의 초기화 오류가 다른 기능까지 멈추지 않도록 보강했습니다.

### v4.2.5 핵심 개선

- 좁은 사이드바에서도 업데이트 확인·실행·나중에 버튼이 줄바꿈되거나 잘리지 않도록 배치를 정리했습니다.
- PDF 편집과 개인정보 검토 화면에서 Windows 정밀 트랙패드 핀치 확대·축소를 지원합니다.

### v4.2.4 주요 개선

- 프로그램 공식 이름을 **곰지의 문서 정리기**로 통일하고 업데이트 확인 버튼과 새 버전 강조 표시를 개선했습니다.
- PDF 편집의 `글자 수정`은 선택한 글자 옆에서 바로 입력하고 실시간으로 결과를 미리 볼 수 있습니다.
- 페이지별 OCR 결과를 유지하고, 이미지·스캔이 섞인 페이지에서는 OCR 실행 여부를 먼저 묻습니다.
- 개인정보 가리기에서 `생년월일` 머리글과 인접한 날짜를 찾고 일반 기간 날짜의 오탐을 줄였습니다.

### PaddleOCR 정식 전환

- 기존 Tesseract OCR을 제거하고 **PaddleOCR PP-OCRv5 한국어 모델**을 정식 엔진으로 통합했습니다.
- 표와 여러 칸이 섞인 증명서·행정 양식에서 글자 영역 검출과 한국어 인식 성능을 개선했습니다.
- PDF 편집과 개인정보 가리기가 같은 OCR 엔진과 좌표 체계를 사용합니다.
- 최초 OCR 사용 시 필요한 모델 약 19MB를 한 번 내려받습니다.

### 개인정보 검토 개선

- 텍스트 레이어가 있는 PDF는 OCR 없이 직접 분석합니다.
- 스캔 인쇄체 OCR은 사용자가 체크한 경우에만 실행하며, 체크 해제가 기본값입니다.
- 자동 검사할 수 없는 이미지·스캔 영역은 빨간 빗금과 안내 문구로 표시합니다. 이 표시는 개인정보를 자동으로 가렸다는 뜻이 아니므로 사용자가 직접 검토해야 합니다.
- 수동 마스킹 영역을 자동 후보와 분리해 목록 최상단에서 확인할 수 있습니다.

### 편집·업데이트 개선

- 덮어쓰기 영역 안에서 글자를 직접 입력하고 글꼴·크기·자간을 실시간으로 미리 볼 수 있습니다.
- 마스킹·텍스트·서명 레이어는 저장 전까지 이동·크기 조절·삭제·모든 페이지 반복이 가능합니다.
- 현재 버전과 업데이트 확인 기능을 메인 화면 좌측 하단에 배치했습니다.
- 업데이트 실패 시 이전 관리 파일을 자동 복구하며, 프로그램 폴더에 사용자가 따로 넣은 파일은 건드리지 않습니다.

자세한 변경 사항은 [v4.3.0 릴리스 노트](https://github.com/conoms/school_docs_release/releases/tag/v4.3.0)를 확인해 주세요.

## 설치 방법

1. [최신 릴리스](https://github.com/conoms/school_docs_release/releases/latest)에서 `school_docs-vX.Y.Z-win64.zip`을 받습니다.
2. 가능하면 압축을 풀기 전에 ZIP 파일을 **우클릭 → 속성 → 차단 해제 → 적용**합니다.
3. 압축을 풀고 `school_docs\school_docs.exe`를 실행합니다.
4. 처음 실행할 때 Windows가 PC를 보호했다는 창이 나오면 **추가 정보 → 실행**을 선택합니다.

> [!IMPORTANT]
> 압축 해제 후 `Python.Runtime.dll` 또는 `Failed to resolve Python.Runtime.Loader.Initialize` 오류가 발생하면 Windows가 ZIP 내부 파일을 차단한 경우가 많습니다. 프로그램 시작 시 `_internal`과 앱 관리 파일만 자동 해제합니다. 그래도 실행되지 않으면 함께 들어 있는 **`school_docs 실행 오류 해결.cmd`**를 한 번 실행하거나, ZIP 속성에서 차단을 해제한 뒤 다시 풀거나, **반디집 등 별도 압축 프로그램**으로 새로 압축 해제해 주세요.

Python을 별도로 설치할 필요는 없습니다. 프로그램 실행에 필요한 Python과 라이브러리는 배포 파일에 포함되어 있습니다.

## 자동 업데이트

- 프로그램 실행 시 GitHub의 최신 정식 버전을 백그라운드에서 확인합니다.
- 좌측 하단의 **업데이트 확인** 버튼으로 언제든지 직접 확인할 수 있습니다.
- 새 버전이 있어도 자동으로 내려받지는 않으며, 사용자가 **지금 업데이트**를 눌러야 진행됩니다.
- ZIP과 SHA-256을 검증한 뒤 기존 `school_docs` 폴더의 프로그램 관리 파일만 교체합니다.
- 새 버전이 정상 실행되지 않으면 백업한 이전 파일을 복구합니다.

`v4.2.2` 이하 버전에는 이 업데이트 기능이 없으므로 최신 버전을 한 번 직접 설치해야 합니다. v4.2.3~v4.2.7은 기존 저장소의 v4.2.8 브리지를 거쳐 v4.3.0 이상으로 이동합니다.

## 로컬 처리와 개인정보 보호

- PDF, 이미지, 명단, OCR 결과는 외부 분석 서버나 LLM으로 전송하지 않습니다.
- OCR과 개인정보 개체 인식은 사용자의 PC에서 실행합니다.
- OCR·개인정보 자동 탐지는 누락이나 오탐이 있을 수 있으므로 저장 전에 검토 창에서 결과를 확인해야 합니다.
- 개인정보 가리기는 단순히 검은 상자를 얹는 방식이 아니라 선택한 내용을 PDF에서 실제 redaction으로 제거합니다.

선택 기능을 처음 사용할 때 다음 로컬 모델을 내려받을 수 있습니다.

| 모델 | 용도 | 대략적인 크기 |
| --- | --- | ---: |
| PaddleOCR PP-OCRv5 | 한국어 인쇄체 OCR | 약 19MB |
| 로컬 개인정보 개체 인식 모델 | 문맥 기반 이름·전화·전자우편 등 후보 보완 | 약 174MB |

모델만 인터넷에서 내려받으며, 사용자의 문서는 모델 제공처로 전송하지 않습니다.

## 알아둘 점

- **손글씨 OCR은 지원하지 않습니다.** 손글씨와 도장 주변은 직접 확인해 주세요.
- 개인정보 자동 탐지는 최종 판단을 대신하지 않습니다. 결과 파일을 저장하기 전에 반드시 후보와 빨간 빗금 영역을 검토해 주세요.
- 오피스 문서의 PDF 변환·일괄 인쇄에는 해당 PC에 Microsoft Excel·Word·PowerPoint가 필요할 수 있습니다.
- 현재 공개 배포본은 Windows x64용입니다.

## 배포 파일 확인

각 릴리스에는 다음 두 파일을 함께 제공합니다.

- `school_docs-vX.Y.Z-win64.zip`: 실행 프로그램
- `school_docs-vX.Y.Z-win64.zip.sha256`: ZIP 파일의 SHA-256 체크섬

Windows 기본 명령으로 체크섬을 확인하려면 압축 파일이 있는 폴더에서 다음 명령을 실행합니다.

```powershell
certutil -hashfile school_docs-v4.3.0-win64.zip SHA256
```

출력된 값이 `.sha256` 파일의 값과 같으면 다운로드 파일이 정상입니다.
